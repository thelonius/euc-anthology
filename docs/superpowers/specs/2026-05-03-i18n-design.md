# i18n + английская локализация — дизайн

**Дата:** 2026-05-03
**Статус:** утверждён, в работе
**Скоуп:** добавить английскую версию сайта с переключателем языка и автоопределением по локали браузера. Перевод выполняется поэтапно: инфраструктура → UI → глоссарий → главы по одной.

## Цель

Сайт **Моноколесо Изнутри** сейчас существует только на русском. Аудитория EUC-сообщества интернациональна — англоязычные читатели не могут воспользоваться материалом. Задача — открыть проект для английской аудитории, сохранив русскую версию первоклассной.

## Не-цели

- Поддержка третьего и далее языков (архитектура совместима, но активно не разрабатывается).
- Машинный перевод. Каждая строка переводится осознанно с учётом инженерной терминологии.
- SSR/pre-rendering. Сайт остаётся клиентским SPA на GitHub Pages.
- Автотесты на полноту переводов. Пропущенные ключи безопасно дают fallback на русский.
- Плюрализация (Intl.PluralRules). Не нужна на текущем контенте, добавится при необходимости.

## Решения и trade-offs

### Гибрид «словарь + параллельные файлы глав»

Контент делится на два слоя:

1. **UI-подложка** (~200 строк): sidebar, кнопки, тултипы, лейблы. Живёт в одном словаре `src/i18n/ui.js`.
2. **Проза глав** (~6.5 тыс. строк) с вкраплённым JSX (`<Term>`, `<code>`, формулы): живёт в параллельных файлах `Prologue.ru.jsx` / `Prologue.en.jsx`.

Альтернативы рассмотрены:

- **Только словарь** для всего: длинные абзацы с JSX превращаются в ад с плейсхолдерами `<0>...</0>`. Отвергнуто.
- **Только параллельные файлы** для всего, без словаря: дублирует константы (`'Скопировано'` в каждом месте). Отвергнуто.
- Гибрид по природе контента: каждый слой получает идиоматичный для него инструмент.

### Ключи словаря — русская строка-источник (gettext-style)

```js
t('Пролог') // → 'Prologue' в EN, 'Пролог' в RU
```

Альтернатива — абстрактные ключи `t('chapter.prologue.label')`. Отвергнуто: на 200 строк это лишний слой косвенности и интеллектуальный труд по придумыванию ключей. Так работают gettext, Django i18n, lingui.

Минус: при переименовании русской строки нужно обновить и вызов, и ключ в `en.js`. Это переживаемо — grep по точной строке надёжен.

### Параллельные файлы глав, не «общий компонент с пропом lang»

Каждая глава — пара файлов:

```
src/chapters/Prologue.jsx        # тонкий роутер
src/chapters/Prologue.ru.jsx     # русская версия (бывший Prologue.jsx, переименован)
src/chapters/Prologue.en.jsx     # английская версия
```

`Prologue.jsx` использует `<LocalizedChapter id="prologue" ru={Ru} en={En}/>` который оборачивает `Suspense` + lazy-импорт + бейдж «Translation in progress».

Альтернатива — единый компонент с `if (lang === 'en') ...` внутри. Отвергнуто: проза переплетена с layout-логикой и canvas-симуляторами, switch-by-language делает код нечитаемым.

### Lazy-loading через `preact/compat`

Главы загружаются по требованию:

```js
const Ru = lazy(() => import('./Prologue.ru'))
const En = lazy(() => import('./Prologue.en'))
```

Vite автоматически разделяет на чанки. Текущий монолитный бандл 1.35 МБ распадётся на UI shell + 15 чанков по главам. При смене языка догружается чанк альтернативной локали — Suspense покажет skeleton.

### URL: query-параметр `?lang=`, не path-сегмент

```
https://thelonius.github.io/euc-anthology/?lang=en#foc
```

Альтернатива — `/en/` в пути. Отвергнуто: требует разделить на два HTML-файла на GitHub Pages, ломает текущий hash-routing. Query-параметр совместим с hash-роутингом и достаточен для `<link rel="alternate" hreflang>`.

При смене языка URL обновляется через `history.replaceState`, не `pushState` — переключение языка не должно засорять историю браузера. Кнопка back ведёт к предыдущей главе.

### Эвристика автоопределения

Приоритеты (сверху вниз):

1. `?lang=` в URL — выигрывает всегда.
2. `localStorage['euc-anthology:lang']` — сохранённый выбор пользователя.
3. `navigator.languages`:
   - префикс `ru`, `uk`, `be`, `kk`, `ky` → `ru`
   - всё остальное → `en`
4. Дефолт — `en`.

`localStorage` доступ обёрнут в try/catch (приватный режим Safari).

### `<html lang>`, `<title>`, мета-теги

Все обновляются JS-ом при смене языка. Изначальный SSR-контент в `index.html` остаётся русским как дефолт для краулеров. В `<head>` добавляются `<link rel="alternate" hreflang="ru|en|x-default">`.

## Архитектура

### Файловая раскладка

```
src/
├── i18n/
│   ├── index.js              # Provider, useT, useLang, useSetLang
│   ├── detect.js             # detectInitialLang, persistLang, sync URL
│   ├── ui.js                 # словарь UI: { en: { 'Пролог': 'Prologue', ... } }
│   ├── chapters.js           # TRANSLATED_TO_EN: Set<string>
│   └── meta.js               # applyMeta(t): обновление title, description, og:*
│
├── chapters/
│   ├── Prologue.jsx          # тонкий роутер: <LocalizedChapter id ru en/>
│   ├── Prologue.ru.jsx       # текущий контент (git mv)
│   ├── Prologue.en.jsx       # перевод (или re-export из .ru до перевода)
│   └── ... × 15 глав
│
├── glossary/
│   ├── glossary.js           # роутер: useGlossaryEntry(term), TERMS_SORTED
│   ├── ru.js                 # текущий glossary.js (git mv)
│   ├── en.js                 # { ENTRIES: { 'FOC': { desc: '...' }, ... } }
│   ├── wrapTerms.js          # без изменений в логике
│   └── Term.jsx              # использует useGlossaryEntry, useT
│
├── components/
│   ├── LanguageToggle.jsx    # RU·EN кнопка в header
│   ├── TranslationPending.jsx # янтарный бейдж сверху главы
│   ├── LocalizedChapter.jsx  # Suspense + lazy + бейдж
│   ├── ChapterLayout.jsx     # без изменений
│   └── ...
│
└── app.jsx                   # обёрнут в I18nProvider, header содержит LanguageToggle
```

### Контракт i18n core

```js
// useT() возвращает функцию (ruKey: string, vars?: object) => string
const t = useT()
t('Пролог')                              // 'Prologue' / 'Пролог'
t('Найдено {n} совпадений', { n: 3 })    // подстановка через {var}

// useLang() / useSetLang()
const lang = useLang()                   // 'ru' | 'en'
const setLang = useSetLang()
setLang('en')                            // обновляет state, localStorage, URL, html.lang, title, meta
```

### Контракт глоссария

```js
// useGlossaryEntry(term) возвращает { full?: string, desc: string } или null
// full берётся из ru.js (общий для обеих локалей)
// desc — из текущей локали с fallback на ru.js
const def = useGlossaryEntry('FOC')
```

### `LocalizedChapter`

```jsx
import { TRANSLATED_TO_EN } from '../i18n/chapters'

export function LocalizedChapter({ id, ru: Ru, en: En }) {
  const lang = useLang()
  const hasEn = TRANSLATED_TO_EN.has(id)
  const showPending = lang === 'en' && !hasEn
  const Component = lang === 'en' && hasEn ? En : Ru
  return (
    <Suspense fallback={<ChapterSkeleton/>}>
      {showPending && <TranslationPending/>}
      <Component/>
    </Suspense>
  )
}
```

## План миграции

### Фаза 0 — Инфраструктура (один большой PR)

- Создать `src/i18n/` модули, `LanguageToggle`, `TranslationPending`, `LocalizedChapter`
- Обернуть `<App>` в `<I18nProvider>`, добавить toggle в header
- Переименовать 15 глав в `*.ru.jsx`, создать тонкие роутеры и заглушки `*.en.jsx` (re-export `Ru`)
- Переименовать `glossary.js` → `ru.js`, создать пустой `en.js`, новый `glossary.js` — роутер
- Обновить `Term.jsx` на хук `useGlossaryEntry`
- Добавить `<link rel="alternate" hreflang>` в `index.html`

**Acceptance:** RU-версия сайта неотличима от текущей. EN-версия показывает русскую прозу с бейджем «Translation in progress», но переводит UI-shell не начинает (это фаза 1).

### Фаза 1 — UI shell

Заменить инлайн-русские строки в `app.jsx`, `Term.jsx`, `LogAnalyzer.jsx`, заголовках секций глав через `t(...)`. Заполнить `i18n/ui.js`.

### Фаза 2 — Глоссарий

Заполнить `glossary/en.js` для всех ~150 терминов.

### Фазы 3..N — Главы по одной

Порядок: Prologue → Hardware → RAM Map → State Machine → ISR Anatomy → Calibration → IMU → Balance → Telemetry → Thermal → FOC → Flux Observer → Field Weakening → HFI → Hack.

Каждая глава = один PR (или две-три родственные вместе). После перевода — добавить id в `TRANSLATED_TO_EN`.

### Фаза N+1 — Финал

- Двуязычный README
- `package.json` description нейтрален или двуязычен
- Lighthouse-проверка

## Acceptance criteria

### Фаза 0

- [ ] Чистый заход без `?lang=` и без localStorage с `navigator.language=ru-RU` → ru
- [ ] То же с `navigator.language=en-US` → en, проза глав ru с бейджем
- [ ] Клик `LanguageToggle` обновляет URL, глава не сбрасывается, history не засоряется
- [ ] Шер-ссылка `?lang=en#foc` → en+FOC независимо от browser-локали
- [ ] `?lang=garbage` → fallback без падения
- [ ] Все 15 глав открываются на ru без визуальных регрессий
- [ ] Главный чанк бандла меньше текущих 1.35 МБ (главы в lazy-чанках)

### Фаза 1

- [ ] Sidebar, header, кнопки, тултипы Term на EN — английские
- [ ] Длина EN-строк не ломает sidebar 220px («Field Weakening», «Translation in progress»)

### Фаза 2

- [ ] Тултип FOC, PWM, Clarke и т.д. в любой главе на EN — английский desc
- [ ] Term без перевода в `en.js` — fallback на ru desc

### По каждой главе

- [ ] Глава на EN читается без вкраплений ru (кроме сознательных: цитаты Пелевина и т.п.)
- [ ] Симулятор/canvas работает идентично ru-версии
- [ ] `wrapTerms` находит термины (FOC, PWM) в английском тексте

## Что НЕ делаем

- Внешние i18n-библиотеки (`react-i18next`, `lingui`, `formatjs`). 50–200 КБ ради того, что умещается в 50 строк.
- Отдельные namespace-ы словаря.
- ICU MessageFormat. `{var}` интерполяции достаточно.
- Cookie-баннер. `localStorage` для UI-языка не tracking.
- Pre-rendering / SSR.
- Sitemap.xml.
- Автотесты на полноту переводов.
- `og:image`. У проекта нет hero-картинки для соцсетей.
- Машинный перевод.
